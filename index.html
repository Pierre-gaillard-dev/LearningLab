Présentation sur les Frameworks



Qu'est ce que c'est un Framework ?


Définition : Un outils qui permet de changer sa manière de coder en ajoutant des fonctions ou encore des syntaxes propre à l'outil choisi. Celui-ci doit correspondre au projet que vous mener.

Exemple : Three js -> 3D en code


Quel différence avec une librairie ?


Principal : Une librairie peut ajouter des fonctions mais ne changera pas la syntaxe ni la structure de votre code.

Pourquoi on utilise un Framework ?


Principal : Pour utiliser une structure adapter à la tâche voulue.

Exemple :

React pour les applications
Tailwind pour être capable de répéter les propriétés

div class=flex items-center justify-center

/div

Comment choisir son Framework ?


Principal : Cela va dépendre de votre but. Cela dépend également de vos connaissances et de vos goûts


Si vous souhaitez expérimenter avec de nouveaux Frameworks alors il faudra vous renseigner sur leur utilisation.

Quels sont les avantages et inconvénients d'utiliser des Frameworks ?


Avantages :


  • La simplicité

  • L'adaptabilité


Inconvénient :


  • L'optimisation

Quelles sont les meilleures pratiques pour développer et maintenir une application basée sur un framework ?


  • Comprenez ses concepts et respectez ses conventions

  • Planifiez une architecture claire

  • Mettez régulièrement à jour le framework

  • Intégrer des tests automatisés

  • Documentez le code

  • Utilisez les dépendances avec prudence

  • Optimisez les performances

  • Appliquez des bonnes pratiques de sécurité spécifiques au framework

Quels sont les différences entre faire du code traditionnel ou avec des Frameworks ?


Code traditionnel :


  • une liberté totale pour structurer et développer

  • Long et sujet à erreurs.



Code Framework :


  • Une structure prédéfinie avec des outils

  • Ajoute des fonctionnalités intégrées accélérant le développement

  • Impose des règles qui peuvent limiter la flexibilité



Le code traditionnel convient mieux aux projets uniques ou légers, tandis que les frameworks sont idéaux pour des projets complexes nécessitant évolutivité, maintenance et collaboration simplifiée.
